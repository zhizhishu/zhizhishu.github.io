Flux 3x-ui Orchestrator

一个基于 Flux Panel 的新主控项目：在一个工作台里选择多台服务器，自动安装/配置 3x-ui，把 VLESS Reality、VMess WS、Trojan TLS、Shadowsocks 和 Snell 放进统一协议节点层，并统一回看 Xray、证书、流量与服务状态。

控制平面拆成清楚的四层

保持 KISS 和 YAGNI：不把 Snell 硬塞进 Xray，而是在主控做统一节点抽象，底层分别适配 3x-ui/Xray API 和 Snell agent/systemd。

Control Server

登记主控/副控服务器，保存 agent token、3x-ui endpoint、API token、证书状态、服务状态、流量和心跳时间。

Protocol Node

把 Xray inbound 和 Snell 服务记录到 `protocol_node`，同一张节点列表里管理创建、删除、重启、同步和运行状态。

3x-ui Proxy

通过 Bearer token 操作 `/panel/api/inbounds/*`，读取 Xray config/outbounds，并可请求远端 Xray 重启。

Deploy Task

主控生成一键编排脚本；授权副控 agent 拉取、执行、回传日志和元数据，避免不可追踪的远程改动。

入站和出站都能从主控发起
