Flux 3x-ui Orchestrator

一个基于 Flux Panel 的新主控项目：把转发面板的 UI 和节点基础，扩展成可以管理多台服务器、生成协议部署任务、接入 3x-ui/Xray 与 Snell 的控制平面。

控制平面拆成清楚的三层

保持 KISS 和 YAGNI：先不硬搬整套 3x-ui，只抽取节点、协议、任务这些最小闭环。

Control Server

登记主控/副控服务器，保存 agent token、心跳时间、Xray/Snell 版本、CPU 和内存状态。

Protocol Profile

将 Snell、VLESS、VMess、Trojan、Shadowsocks 作为可下发的模板，而不是散落在页面和脚本里。

Deploy Task

先生成可审计任务和脚本，再交给 agent 执行，避免主控直接对服务器做不可追踪的远程改动。

部署任务先生成 runbook

Snell 支持来自脚本能力提取，但第一版默认只生成非交互脚本，不自动登录服务器执行。

接下来做真正的编排

下一步从“能生成任务”推进到“agent 可以执行、上报、同步远端 3x-ui inbound”。

  1. Agent executor 副控拉取任务、执行脚本、回传日志和状态，主控只保留审计链。
  2. 3x-ui remote client 封装 Bearer token、basePath、探活、inbound 创建和流量快照同步。
  3. Protocol safety 主控解析 Snell 版本、下载地址和 checksum，减少 agent 侧动态抓网页的不确定性。
  4. Operational dashboard 把服务器健康、部署历史、协议库存和异常日志放到同一个工作台。
