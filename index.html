Flux 3x-ui Orchestrator

一个基于 Flux Panel 的新主控项目：把转发面板的 UI 和节点基础，扩展成可以管理多台服务器、自动执行 Snell 部署任务，并直接代理 3x-ui/Xray 入站、出站与流量同步的控制平面。

控制平面拆成清楚的四层

保持 KISS 和 YAGNI：不硬搬整套 3x-ui，而是用主控代理连接真实面板 API，再逐步补结构化表单和流量同步。

Control Server

登记主控/副控服务器，保存 agent token、心跳时间、3x-ui endpoint、basePath、API token 和可选登录凭据。

3x-ui Proxy

通过 Bearer token 操作 `/panel/api/inbounds/*`，读取 Xray config/outbounds，并可请求远端 Xray 重启。

Outbound Session

对 3x-ui `/panel/xray/update` 使用登录 + CSRF 通道保存完整 Xray/outbound JSON，避免绕开面板规则。

Deploy Task

Snell 先生成可审计脚本，再由授权副控 agent 拉取、执行并回传日志，避免主控直接做不可追踪的远程改动。

入站和出站都能从主控发起

入站走 3x-ui Bearer API，出站读取从运行配置提取，保存则使用登录态保护的 Xray 模板接口。

接下来做更顺手的编排

核心链路已经落地，后续重点是把运维体验做扎实：常驻 agent、定时同步、协议校验和运行态看板。

  1. Agent service 给 `scripts/flux-agent.sh` 补 systemd unit 和安装脚本，让副控可常驻、可重启、可审计。
  2. Scheduled traffic sync 把手动“同步流量”升级为周期任务，支持跨服务器历史对比。
  3. Protocol guardrails 增加 Reality key、端口占用、outbound tag 和 Snell checksum 校验。
  4. Operational dashboard 把服务器健康、部署历史、协议库存和异常日志放到同一个工作台。
