Flux 3x-ui Orchestrator

一个基于 Flux Panel 的新主控项目：把转发面板的 UI 和节点基础，扩展成可以管理多台服务器、生成 Snell 部署任务，并直接代理 3x-ui/Xray 入站与出站配置的控制平面。

主控已经接上 3x-ui 面板 API

当前版本不只停在骨架：已经可以配置远端 3x-ui 面板、同步 inbound、提交 inbound 操作、读取 outbounds，并保留 Snell 非交互部署任务。

控制平面拆成清楚的四层

保持 KISS 和 YAGNI：不硬搬整套 3x-ui，而是用主控代理连接真实面板 API，再逐步补结构化表单和流量同步。

Control Server

登记主控/副控服务器，保存 agent token、心跳时间、3x-ui endpoint、basePath、API token 和可选登录凭据。

3x-ui Proxy

通过 Bearer token 操作 `/panel/api/inbounds/*`，读取 Xray config/outbounds，并可请求远端 Xray 重启。

Outbound Session

对 3x-ui `/panel/xray/update` 使用登录 + CSRF 通道保存完整 Xray/outbound JSON，避免绕开面板规则。

Deploy Task

Snell 仍先生成可审计脚本，再交给后续 agent 执行，避免主控直接做不可追踪的远程改动。

入站和出站都能从主控发起

入站走 3x-ui Bearer API，出站读取从运行配置提取，保存则使用登录态保护的 Xray 模板接口。

POST /api/v1/three-xui/inbounds/add

POST /api/v1/three-xui/outbounds

接下来做更顺手的编排

现在已经能代理 3x-ui 出入站，下一步是把 JSON 操作升级成结构化表单，并让副控 agent 自动执行 Snell 任务。

  1. Structured inbound forms 把 VLESS Reality、VMess WS、Trojan TLS、Shadowsocks 的 JSON payload 做成安全表单。
  2. Traffic sync 把远端 inbound/client/outbound 流量快照同步进主控本地表，支持跨服务器对比。
  3. Agent executor 副控拉取 Snell 任务、执行脚本、回传日志和状态，主控保留审计链。
  4. Operational dashboard 把服务器健康、部署历史、协议库存和异常日志放到同一个工作台。
